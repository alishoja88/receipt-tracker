## ⚠️ جریان مدیریت خطا

```
┌─────────────────────────────────────────────────────────────┐
│                    خطا از سرور می‌آید                        │
└──────────────────────┬──────────────────────────────────────┘
                       │
                       ▼
            ┌──────────┴──────────┐
            │                     │
            ▼                     ▼
    ┌──────────────┐      ┌──────────────┐
    │   401 (Unauthorized)│  خطاهای دیگر  │
    └──────┬───────┘      └──────┬───────┘
           │                     │
           │                     ▼
           │         ┌───────────────────────────┐
           │         │  چک می‌کند:                │
           │         │  • آیا endpoint auth است؟ │
           │         │  • آیا _retry = true است؟ │
           │         └───────────┬───────────────┘
           │                     │
           │                     ▼
           │         ┌───────────────────────────┐
           │         │  تبدیل به ApiError:       │
           │         │                           │
           │         │  اگر errorData.error:     │
           │         │    → ApiError با code      │
           │         │                           │
           │         │  اگر errorData.message:   │
           │         │    → ApiError با message  │
           │         │                           │
           │         │  اگر network error:       │
           │         │    → NETWORK_ERROR        │
           │         │                           │
           │         │  در غیر این صورت:         │
           │         │    → INTERNAL_SERVER_ERROR│
           │         └───────────┬───────────────┘
           │                     │
           │                     ▼
           │         ┌───────────────────────────┐
           │         │  throw ApiError           │
           │         │  (خطا به catch می‌رود)     │
           │         └───────────────────────────┘
           │
           ▼
┌─────────────────────────────────────────────────────────────┐
│  اگر 401 و endpoint auth نیست:                              │
│                                                             │
│  1. _retry = true (جلوگیری از حلقه)                        │
│  2. چک می‌کند: آیا در حال refresh هستیم؟                   │
│     • اگر بله: صبر می‌کند                                   │
│     • اگر نه: شروع refresh می‌کند                           │
│  3. بعد از refresh:                                         │
│     • توکن جدید به header اضافه می‌شود                      │
│     • درخواست دوباره زده می‌شود                             │
│                                                             │
│  اگر refresh ناموفق باشد:                                   │
│     • کاربر logout می‌شود                                   │
│     • به /login هدایت می‌شود                                │
└─────────────────────────────────────────────────────────────┘
```

## 💡 نکات کلیدی

### ✅ چیزهایی که خودکار انجام می‌شود:
1. ✅ اضافه کردن توکن به همه درخواست‌ها
2. ✅ refresh کردن توکن وقتی منقضی می‌شود
3. ✅ جلوگیری از refresh چندگانه همزمان
4. ✅ تبدیل خطاها به فرمت استاندارد
5. ✅ لاگ کردن درخواست‌ها و پاسخ‌ها (در dev mode)

### ⚠️ چیزهایی که باید خودتان انجام دهید:
1. ⚠️ مدیریت خطا در catch block
2. ⚠️ نمایش پیام خطا به کاربر
3. ⚠️ هدایت کاربر به صفحه login (اگر نیاز باشد)

## 📝 مثال عملی کامل

```typescript
// در کامپوننت React
const ReceiptList = () => {
  const [receipts, setReceipts] = useState<Receipt[]>([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    const fetchReceipts = async () => {
      try {
        setLoading(true);
        
        // این یک خط ساده است، اما پشت صحنه چه اتفاقی می‌افتد؟
        const response = await api.get<ApiPaginatedResponse<Receipt>>(
          '/api/receipts',
          { page: 1, limit: 20 }
        );
        
        // اگر به اینجا رسیدیم، یعنی:
        // ✅ درخواست موفق بوده
        // ✅ توکن معتبر بوده (یا refresh شده)
        // ✅ داده‌ها آماده هستند
        
        setReceipts(response.items);
        setError(null);
      } catch (err) {
        // اگر به اینجا رسیدیم، یعنی:
        // ❌ خطایی رخ داده
        // ❌ خطا به ApiError تبدیل شده
        
        if (err instanceof ApiError) {
          // می‌توانیم نوع خطا را چک کنیم
          if (err.code === ErrorCode.UNAUTHORIZED) {
            // کاربر باید login کند
            navigate('/login');
          } else {
            // خطای دیگر - نمایش پیام
            setError(err.getLocalizedMessage());
          }
        } else {
          setError('خطای ناشناخته');
        }
      } finally {
        setLoading(false);
      }
    };

    fetchReceipts();
  }, []);

  // ...
};
```

**چه اتفاقی در پشت صحنه می‌افتد؟**

```
1. api.get() فراخوانی می‌شود
   ↓
2. Request Interceptor:
   • توکن از store: "xyz123..."
   • Header: Authorization: Bearer xyz123...
   ↓
3. درخواست: GET /api/receipts?page=1&limit=20
   ↓
4. سرور پاسخ می‌دهد: 200 OK
   ↓
5. Response Interceptor:
   • لاگ می‌شود
   • پاسخ برمی‌گردد
   ↓
6. api.get() پردازش می‌کند:
   • response.data را برمی‌گرداند
   ↓
7. داده به کامپوننت برمی‌گردد:
   • response.items = [...receipts]
   • response.pagination = {...}
```
